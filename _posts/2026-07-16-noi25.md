## D1P2. Sequence

Firstly, note that an operation is just picking an index $i \in [1,n)$, and then "pulling" the smaller element between $i$ and $i+1$ to be zero, and the larger element to be subtracted correspondingly.

Suppose that you want to make some subarray $A[l..r]$ to become zeroes while keeping $A[l-1]$ and $A[r+1]$ nonzero. How to check if this is possible?

You pick the first index to operate, let's suppose it is $i \in [l-1,r]$. Then, the only way to make stuff work is to "propagate to both sides", i.e. you do the operations $i,i-1,i-2,\ldots,l-1,i+1,i+2,\ldots,r$ in order (you still have to do the checking).

If you write a brute force, you will realise that for all working starting indices $i$, there are only at most $2$ possible resulting pairs of values that $(A[l-1],A[r+1])$ can be: specifically, it only depends on $i \pmod 2$.
